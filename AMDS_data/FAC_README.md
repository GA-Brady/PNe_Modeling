# FAC (Flexible Atomic Code) in Julia via PythonCall

This documents how to set up PFAC (FAC's Python interface) and call it from Julia using PythonCall on macOS arm64.

---

## Prerequisites

- macOS arm64 (Apple Silicon)
- [Conda](https://docs.conda.io/en/latest/miniconda.html) (Miniconda or Miniforge)
- Julia with [juliaup](https://github.com/JuliaLang/juliaup)
- Xcode command line tools: `xcode-select --install`
- gfortran (via conda, see below)

---

## Step 1: Create a Conda environment for FAC

```bash
conda create -n fac_env python=3.10
conda activate fac_env
conda install -c conda-forge gfortran gsl
```

---

## Step 2: Build FAC from source

Clone FAC somewhere permanent outside your project — treat it like a system dependency, not project code. A good location is `~/Repos/FAC/fac` or `~/software/fac`.

```bash
git clone https://github.com/flexible-atomic-code/fac.git ~/Repos/FAC/fac
cd ~/Repos/FAC/fac
```

Configure and build, pointing at the conda env Python:

```bash
conda activate fac_env
./configure PYTHON=$(which python)
make
```

The `make` step will produce warnings like:

```
ld: warning: object file (...) was built for newer macOS version (15.0) than being linked (11.0)
```

These are harmless — ignore them.

### Install PFAC into the conda env

The standard `make install-pfac` target hardcodes `--prefix=/usr/local` which fails without root. Instead, run pip directly with the conda env active:

```bash
pip install .
```

Verify the install worked:

```bash
python -c "from pfac.fac import *; print('PFAC OK')"
```

You should see some initialization numbers printed followed by `PFAC OK`.

---

## Step 3: Set up the Julia project

Create your Julia project directory and activate it:

```bash
mkdir ~/projects/my-fac-project
cd ~/projects/my-fac-project
julia --project=.
```

In the Julia REPL, install PythonCall:

```julia
] add PythonCall
```

---

## Step 4: Point PythonCall at the FAC conda environment

By default PythonCall creates its own internal conda/pixi environment and won't see PFAC. You need to point it at `fac_env`.

Create a `LocalPreferences.toml` file in your project root:

```toml
[CondaPkg]
backend = "System"

[PythonCall]
exe = "/Users/YOUR_USERNAME/conda/envs/fac_env/bin/python"
```

Replace `YOUR_USERNAME` with your actual username. Find the exact path with:

```bash
conda activate fac_env && which python
```

**Important:** these env vars must be set before Julia starts — setting them inside Julia after `using PythonCall` has no effect. The `LocalPreferences.toml` file handles this automatically.

Alternatively, set them in your shell before launching Julia:

```bash
export JULIA_CONDAPKG_BACKEND="System"
export JULIA_PYTHONCALL_EXE="/Users/YOUR_USERNAME/conda/envs/fac_env/bin/python"
julia --project=.
```

---

## Step 5: Verify the setup

In Julia:

```julia
using PythonCall

# Confirm the correct Python is being used
pyimport("sys").executable
# Should print: '/Users/YOUR_USERNAME/conda/envs/fac_env/bin/python'

# Load PFAC
fac = pyimport("pfac.fac")
rfac = pyimport("pfac.rfac")
```

Run a minimal test calculation:

```julia
fac = pyimport("pfac.fac")

fac.SetAtom("Fe")
fac.Config("1s2 2s2 2p6", group="ne_like")
fac.OptimizeRadial(["ne_like"])
fac.Structure("fe.lev.b", ["ne_like"])
fac.PrintTable("fe.lev.b", "fe.lev", 1)
```

## Project layout

```
my-fac-project/
├── Project.toml          # Julia dependencies (auto-generated)
├── Manifest.toml         # Julia lockfile (auto-generated)
├── LocalPreferences.toml # PythonCall config — points at fac_env
├── src/
│   └── your Julia code
├── scripts/
│   └── example calculations (.jl)
└── data/
    └── FAC output files (.lev, .tr, etc.)
```

The FAC source tree (`~/Repos/FAC/fac`) lives outside this project and is not committed here.

## Known issues and workarounds

### `make install-pfac` fails with Permission denied
The Makefile passes `--prefix=/usr/local` to pip, which requires root. Bypass it with:
```bash
pip install .
```

### `'init.h' file not found` or `'sysdef.h' file not found` during build
These headers are generated by `./configure` and compiled by `make`. If they're missing, the build order is wrong. Always run:
```bash
./configure PYTHON=$(which python)
make
pip install .
```
in that exact order. If a previous attempt left partial state, run `make distclean` first.

### PythonCall uses wrong Python despite setting env vars inside Julia
Env vars set inside Julia after `using PythonCall` are too late — PythonCall initializes on first load. Always use `LocalPreferences.toml` or set the vars in the shell before launching Julia.

### macOS deployment target warnings during `make`
```
ld: warning: object file was built for newer macOS version (15.0) than being linked (11.0)
```
Harmless. Suppress with `export MACOSX_DEPLOYMENT_TARGET=15.0` before `make` if desired.

## Important limitations

- **PFAC is not thread-safe.** Never call PFAC functions from multiple Julia threads or concurrent tasks. Only one calculation can run in one Python kernel at a time.
- **Do not use `Threads.@spawn` or `@async`** to parallelize PFAC calls.

## Available PFAC modules

| Module | Purpose |
|---|---|
| `pfac.fac` | Main atomic structure calculations |
| `pfac.crm` | Collisional-radiative model |
| `pfac.pol` | Polarization calculations |
| `pfac.rfac` | Read FAC binary output files |
| `pfac.atom` | Higher-level atom helpers |
| `pfac.util` | Utilities |

Example scripts for atomic structure, excitation, photoionization, autoionization, and more are in the `demo/` directory of the FAC source tree.
